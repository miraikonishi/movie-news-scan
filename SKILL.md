# movie-news-scan

ハリウッド映画ニュースを朝に収集し、**日本の一般の映画ファンに響く候補**を10〜15本に絞って提示するスキル。収集だけを担う。記事執筆・Googleシートへの書き込み・本文取得（Firecrawl等）は**このスキルの範囲外**。

## When to Use

- 「今日のニュースを探して」「映画ニュース」「朝のスキャン」と言われたとき
- 毎朝の定期実行（cron）で映画ニュース候補を作るとき

## What this skill does NOT do（境界）

- ❌ 記事を書かない（候補提示で止まる。執筆は別スキル/別工程）
- ❌ Googleシートに書き込まない（シートは**読み取り専用**で重複・被りチェックにのみ使う）
- ❌ ソース本文の全文取得はしない（RSSの概要までで判断する）
- ❌ 候補から自動で4本に確定しない（最終選定は人間。Hermesは候補リストを出すまで）

## Prerequisites

- `node`（標準モジュールのみ使用。追加パッケージ不要）
- `bash`, `curl`
- インターネット接続（RSS取得＋GoogleシートのCSVエクスポート取得）

## Inputs

なし。RSSは自動で全取得する。推測で数本だけ挙げてはいけない。**必ずスクリプトを回す。**

## Outputs

候補リスト（10〜15本）。各候補に:
- 日本語タイトル（和訳）
- URL
- ソース媒体
- おすすめ理由（1行）
- スコア（下記基準）

## Procedure

1. このスキルディレクトリで収集スクリプトを実行する:
   ```bash
   bash "$SKILL_DIR/scripts/scan-news.sh"
   ```
   - `$SKILL_DIR` はこの SKILL.md があるディレクトリ（Mac mini では通常 `~/.hermes/skills/movie-news-scan`）。
   - スクリプトは内部で `fetch-news.cjs`（RSS取得・過去48時間・重複除去）と `check-sheet.sh`（Googleシート確認）を順に呼ぶ。出力は `scripts/output/rss-latest.json` にも保存される。
2. 取得結果（RSS一覧）と Googleシートの「政氏さん担当中」リストを突き合わせる。
3. 下記スコアリング基準で各記事を採点する。
4. **政氏さん担当済みは除外**（シートで確認。被りは出さない）。
5. スコア上位を **10〜15本** 提示する。3〜4本で済ませない。
6. 人間が4本を選ぶまで待機する（1日4本が前提）。

## Scoring（日本の読者基準）

- **+3** 大型フランチャイズ（マーベル、スター・ウォーズ、ボンド、ジュラシック、ゾンビランド等）
- **+3** 有名スター（ディカプリオ、トム・クルーズ、ノーラン等、日本で名の通った監督・俳優）
- **+3** 日本関連（日本市場・ジブリ・日本人クリエイター・日本ロケ等）
- **+2** 業界トレンド（ストリーミング、AI、興収の歴代記録更新）
- **+1** 複数メディアで報じられている（注目度が高い証拠）
- **-2** アメリカ国内だけの話題・純粋なゴシップ
- **-3** 週間興行ランキング報道（映画.comが毎週出すため除外。ただし歴代記録更新などの記録的数字、興収に関する分析・エピソード記事は対象）
- **-5** 政氏さん担当済み（実質除外）

## Source priority

**Deadline > Variety > THR > Google News**。同じネタが複数媒体に出ていたら注目度が高い証拠で、ソースは元ネタになりがちな Deadline を優先する。

## Rules

- 自分の記憶やWeb検索だけで候補を作らない。**必ず scan スクリプトを回す。**
- 業界ニュース（合併・買収・規制・人事等）は、英文のままだと日本の読者に遠い。「観客・ファンにとって何が変わるか」で価値を判断してスコアに反映する。
- 邦題・固有名詞の細かい表記はこのスキルの範囲外（執筆工程の担当）。候補段階では和訳は分かりやすさ優先でよい。
